## Project Overview

**Incident Helper** — a ServiceNow scoped application (`x_1470706_incident`) built with the ServiceNow Now SDK (`@servicenow/sdk@4.7.2`) using the Fluent DSL. Fluent is a TypeScript-based DSL for defining ServiceNow metadata artifacts (tables, business rules, client scripts, ACLs, flows, etc.) as code instead of through the platform UI.

## Commands

```bash
npm run build      # Compile fluent source files; validates syntax and reports errors
npm run deploy     # Push built artifacts to the connected ServiceNow instance (requires auth)
npm run transform  # Convert existing instance XML artifacts into fluent source files
npm run types      # Fetch TypeScript type definitions from the connected instance (for IDE autocompletion)
```

Additional SDK commands (not in package.json scripts):
```bash
npx now-sdk auth --list                        # Check existing instance credentials
npx now-sdk auth --add <url> --type basic      # Add instance credentials interactively
npx now-sdk auth --use <alias>                 # Set default instance
npx now-sdk clean                              # Remove build output and cached artifacts
npx now-sdk pack                               # Package app into a ZIP (update set XML + manifest)
npx now-sdk download                           # Download specific records from an instance
```

## Architecture

### File Layout

```
src/fluent/           # Fluent DSL source (.now.ts files) — define all ServiceNow metadata here
  generated/
    keys.ts           # Auto-generated: maps logical keys to sys_ids for sys_module records
src/server/           # Server-side JS/TS modules referenced by fluent definitions
now.config.json       # App metadata: scope name, scopeId, display name (no instance credentials)
dist/app/             # Build output: XML update set files ready for deployment
target/               # Packaged ZIP artifact
.now/bom.json         # CycloneDX bill-of-materials (auto-generated by build)
```

### Fluent DSL Conventions

- All metadata definitions live in `.now.ts` files under `src/fluent/`. Organize by type (e.g., `business-rules/`, `tables/`, `client-scripts/`).
- Every `.now.ts` file must `import '@servicenow/sdk/global'` to access the `Now` global namespace.
- Artifact types (`BusinessRule`, `Table`, `ClientScript`, `Acl`, etc.) are imported from `@servicenow/sdk/core`.
- Server-side scripts are written as TypeScript/JS functions in `src/server/` and referenced via the `script:` property in fluent definitions — never inline strings.
- Record IDs (`$id`) use `Now.ID['key-name']` — keys are declared in `src/fluent/generated/keys.ts` and must be registered there before use.
- The scope prefix `x_1470706_incident` must be used for all table names, script includes, and other scoped artifacts.

### Auth & Instance Connection

Instance credentials are stored outside the repo in `.now-sdk/` (gitignored) and managed via `npx now-sdk auth`. The `now.config.json` file contains only app metadata, never credentials.

For CI/CD, set environment variables: `SN_SDK_NODE_ENV=SN_SDK_CI_INSTALL`, `SN_SDK_INSTANCE_URL`, `SN_SDK_USER`, `SN_SDK_USER_PWD` (basic auth) or OAuth equivalents.

### Build → Deploy Workflow

1. Write or modify `.now.ts` files in `src/fluent/`
2. `npm run build` — compiles to XML in `dist/app/update/`
3. `npm run deploy` — pushes XML to the instance
4. Use `npm run transform` to pull existing instance artifacts into fluent source when converting existing records
